# 7단계 플랜 검수 파이프라인

**입력**: plan.md (기능 개발 계획서, 전략 문서, 구현 계획)
**출력**: 7명이 순차 검토한 이슈 목록 + 최종 APPROVE/REVISE/REJECT 판정
**목표**: 코딩을 시작하기 전에 계획의 허점을 전부 잡아내는 것

## 시작하기 전

반드시 `plan-review-refs/review-protocol.md` 를 먼저 읽어라.

## 이것은 코드 리뷰가 아니다

```
❌ 코드를 grep으로 스캔하는 것
❌ 기존 코드베이스를 전수 조사하는 것
❌ npm audit, ESLint를 돌리는 것

✅ plan.md 문서를 읽는 것
✅ "이 계획대로 빌딩하면 어떤 문제가 생길까?"를 예측하는 것
✅ "이 계획에서 빠진 것은 무엇인가?"를 찾는 것
✅ "이 계획을 더 좋게 만들 아이디어는?"를 제안하는 것
```

## 파이프라인

```
[plan.md 입력]
    │
    ▼
[1차 보안 감사] "이 계획에 보안 허점이 있는가?"
    │ findings 전달
    ▼
[2차 백엔드 감사] "이 계획의 API·DB 설계에 문제가 있는가?"
    │ findings 누적 전달
    ▼
[3차 프론트엔드 감사] "이 계획의 UI 구현에 리스크가 있는가?"
    │ findings 누적 전달
    ▼
[4차 UX 감사] "이 계획이 사용자 경험을 해치지 않는가?"
    │ findings 누적 전달
    ▼
[5차 아키텍처 감사] "이 계획이 기존 아키텍처와 충돌하지 않는가?"
    │ findings 누적 전달
    ▼
[6차 비즈니스 감사] "이 계획이 매출/성장에 기여하는가?"
    │ findings 누적 전달
    ▼
[7차 최종 판관] "1~6차 종합, 이 계획을 승인할 수 있는가?"
    │
    ▼
[APPROVE / REVISE / REJECT]
```

## 실행 방법

사용자가 plan.md를 제시하면:

1. plan.md 전문을 읽는다
2. 프로젝트의 CLAUDE.md가 있으면 함께 읽어 프로젝트 맥락을 파악한다
3. 1차부터 7차까지 순서대로 각 에이전트 SKILL.md를 로드하여 해당 관점으로 검토한다
4. 각 단계 완료 시 사용자에게 진행 상황을 보고한다
5. 7차 완료 시 최종 판정과 수정된 plan.md를 제시한다

## 누적 컨텍스트 전달

각 에이전트는 이전 에이전트의 발견을 **누적으로** 받는다:

```
2차 백엔드: "1차 보안에서 '인증 계획이 불명확하다'고 지적했는데, 
            API 설계를 보니 실제로 어떤 엔드포인트에 인증이 필요한지 명시가 없다."

4차 UX: "3차 프론트에서 '로딩 상태 계획 없음'을 지적했고,
         2차 백엔드에서 '이 API는 3초+ 걸릴 수 있다'고 지적했는데,
         사용자가 3초 동안 빈 화면을 보게 되는 시나리오가 계획에 없다."
```

이렇게 후순위 에이전트일수록 더 넓은 시야를 갖는다.

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. {구체적 이슈}

🟠 중요한 허점 (빌딩 전 반드시 보완):

🟡 개선 제안 (더 좋은 계획을 위해):

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 최종 판정

```
🟢 APPROVE — 이 계획대로 빌딩하라. 치명적 누락 0, 주요 허점 0.
🟡 REVISE  — 계획 수정 후 재검토. 치명적 0이지만 주요 허점 있음.
🔴 REJECT  — 계획 재수립 필요. 치명적 누락 발견.
```
