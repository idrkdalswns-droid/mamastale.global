# /review — AI 코드 리뷰 (Sanity 3단계 리뷰)

> Quigley: "감정적 애착이 없으니 오히려 더 객관적으로 리뷰한다"

현재 변경사항에 대해 보안 엔지니어 관점으로 코드 리뷰를 수행합니다.

## 체크리스트

### 1단계: 보안 리뷰
- [ ] XSS 취약점 (sanitizeText 사용 여부)
- [ ] SQL 인젝션 (Supabase RLS 우회 가능성)
- [ ] 인증 바이패스 가능성
- [ ] 환경변수 하드코딩 여부
- [ ] Rate limiting 적용 여부

### 2단계: 아키텍처 리뷰
- [ ] Edge Runtime 호환성
- [ ] 500줄 이상 파일 여부
- [ ] 순환 의존성
- [ ] 에러 처리 누락

### 3단계: 품질 리뷰
- [ ] TypeScript any 사용
- [ ] 미사용 import
- [ ] console.log (error/warn 외)
- [ ] 하드코딩된 문자열 (i18n 키 사용 필요)

## 실행 방법

1. `git diff --staged` 또는 `git diff` 확인
2. 변경된 파일 각각 읽기
3. 위 체크리스트 기준으로 리뷰
4. 발견된 이슈를 심각도(🔴🟠🟡)별로 정리
5. 수정 제안 포함
