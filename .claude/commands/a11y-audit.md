# /a11y-audit — 접근성 감사 (WCAG 2.1 AA)

mamastale 전체 또는 특정 페이지의 접근성을 감사합니다.

## 감사 범위

인자가 있으면 해당 파일/경로만, 없으면 전체 컴포넌트를 검사합니다.

## 6단계 검사

### 1단계: 시맨틱 HTML
- [ ] `<nav>`, `<main>`, `<section>`, `<article>` 올바른 사용
- [ ] 헤딩 계층 (h1→h2→h3) 건너뛰기 없음
- [ ] `<button>` vs `<a>` 올바른 구분 (클릭 동작 vs 네비게이션)
- [ ] 리스트 항목에 `<ul>/<ol>` 사용

### 2단계: ARIA 속성
- [ ] `aria-label` 또는 `aria-labelledby` (아이콘 버튼, 모달)
- [ ] `aria-current="page"` (활성 네비게이션)
- [ ] `aria-live="polite"` (동적 콘텐츠 — 채팅 메시지, 토스트)
- [ ] `role="dialog"` + `aria-modal="true"` (모달/시트)
- [ ] `aria-expanded` (드롭다운, 아코디언)

### 3단계: 키보드 네비게이션
- [ ] 모든 인터랙티브 요소에 포커스 가능
- [ ] Tab 순서가 시각적 순서와 일치
- [ ] Escape로 모달/오버레이 닫기
- [ ] 포커스 트랩: 모달 내부에서 Tab 순환
- [ ] `tabIndex` 남용 없음 (0 또는 -1만 사용)

### 4단계: 색상 & 대비
- [ ] 텍스트 대비 비율 ≥ 4.5:1 (일반 텍스트)
- [ ] 큰 텍스트 대비 비율 ≥ 3:1 (18px+ 또는 14px bold)
- [ ] 다크 모드에서도 대비 기준 충족
- [ ] 색상만으로 정보 전달하지 않음 (에러 = 빨강 + 아이콘/텍스트)
- [ ] mamastale 커스텀 색상 검증: cream 배경 위 brown-pale 텍스트 대비

### 5단계: 터치 & 모바일
- [ ] 터치 타겟 최소 44x44px (WCAG 2.5.5)
- [ ] 제스처 대안 제공 (스와이프 → 버튼 대체)
- [ ] DIY PopupBookViewer: 스와이프 외 페이지 이동 버튼 존재
- [ ] `prefers-reduced-motion` 미디어 쿼리 대응

### 6단계: 콘텐츠 접근성
- [ ] 이미지에 `alt` 텍스트 (장식 이미지는 `alt=""`)
- [ ] DIY 동화 이미지: 의미 있는 alt 텍스트
- [ ] 폼 입력에 `<label>` 연결
- [ ] 에러 메시지가 관련 입력과 연결 (`aria-describedby`)

## 실행 방법

1. 대상 파일 또는 전체 `src/components/` 스캔
2. 위 체크리스트 기준 검사
3. 발견된 이슈를 심각도별 정리:
   - 🔴 **차단**: 스크린리더 사용 불가
   - 🟠 **중요**: 접근성 저하
   - 🟡 **개선**: 더 나은 UX 가능
4. 자동 수정 가능한 항목은 바로 수정 제안
