# 캐릭터 디자인 상세 가이드

## 형태 언어(Shape Language) 원칙 — 몰리 뱅 기반

### 기본 형태와 감정 반응

| 형태 | 감정 반응 | 캐릭터 적용 |
|------|----------|-----------|
| 원/곡선 | 안전, 친근, 따뜻, 포용 | 주인공, 엄마/아빠, 친한 친구 |
| 삼각형/뾰족 | 위협, 긴장, 날카로움, 에너지 | 장애물, 까칠한 캐릭터, 경쟁자 |
| 사각형/직선 | 안정, 신뢰, 무게감, 고집 | 어른, 건물, 규칙을 상징하는 존재 |
| 불규칙 | 혼란, 재미, 예측불가 | 장난꾸러기, 마법 존재, 혼돈 상황 |

### 크기와 위치의 심리적 효과

- **큰 캐릭터**: 힘, 중요성, 위협(맥락에 따라)
- **작은 캐릭터**: 약함, 겸손, 귀여움, 공감 유발
- **화면 하단**: 안정, 무거움, 현실감
- **화면 상단**: 자유, 승리, 행복감
- **화면 중앙**: 자신감, 주목
- **화면 가장자리**: 불안, 소외, 모험 시작

### 실루엣 테스트

모든 캐릭터는 검은 실루엣으로 봤을 때도 누구인지 즉시 식별 가능해야 한다.

**식별 요소 체크리스트:**
- 머리/몸의 고유한 비율
- 독특한 부속물 (뿔, 날개, 꼬리 형태)
- 자세/포즈의 특징
- 크기 차이 (다른 캐릭터 대비)

실루엣이 비슷한 캐릭터가 2명 이상이면 하나를 수정해야 한다.

## 연령별 캐릭터 복잡도 상세 스펙

### 영아(0~2세) 캐릭터

**형태:**
- 기본 도형 2~3개의 조합으로만 구성
- 몸 전체가 하나의 큰 원 또는 타원
- 팔다리는 짧은 원통형 또는 생략 가능
- 손가락, 발가락 등 세부 묘사 불필요

**색상:**
- 단일 톤 + 보조 1색 (최대 3색)
- 고채도, 원색 계열
- 패턴/그라디언트 불필요

**표정:**
- 눈과 입만으로 표현 (눈썹 불필요)
- 눈: 크고 동그란 점 또는 반달
- 입: 미소(U자) 또는 놀람(O자)
- 표정 변화 3단계: 기쁨-슬픔-놀람

**예시 스펙:**
```
영아용 고양이 캐릭터:
- 형태: 큰 원(머리) + 작은 원(몸) + 삼각형 2개(귀)
- 색: 주황(전체) + 흰색(배)
- 눈: 큰 검은 점 2개
- 코: 작은 핑크 삼각형
- 특징: 수염 3쌍(단순 직선)
- 총 요소 수: 8개 이하
```

### 유아(3~4세) 캐릭터

**형태:**
- 기본 도형 4~6개 조합
- 머리와 몸이 구분되되, 곡선으로 연결
- 팔다리 표현, 손은 장갑형(손가락 미분리 또는 3~4개)
- 1~2개의 고유 장식/소품 추가 가능

**색상:**
- 주색 + 보조 2색 + 강조 1색 (최대 6색)
- 조화로운 배색, 보색 1쌍까지 허용
- 단순한 패턴 1개 가능 (줄무늬, 물방울)

**표정:**
- 눈 + 눈썹 + 입으로 표현
- 5단계: 기쁨-슬픔-놀람-화남-걱정
- 볼 터치(핑크) 가능
- 땀방울, 느낌표 등 이모티콘적 표현 가능

### 초저(5~7세) 캐릭터

**형태:**
- 자유로운 곡선 조합
- 해부학적으로 더 정확한 비율 (단, 여전히 만화적 과장)
- 손가락 5개, 의상 디테일, 헤어스타일 등
- 소품, 배경과의 상호작용이 복잡해질 수 있음

**색상:**
- 7~10색, 미묘한 톤 변화 가능
- 그라디언트, 질감 표현 가능
- 조명에 따른 색 변화 가능

**표정:**
- 복합 감정 표현 가능 (슬프지만 용감한, 화나지만 참는)
- 미세한 표정 변화 (입꼬리의 미세한 각도 차이)
- 시선 방향이 감정/의도를 전달

## AI 캐릭터 일관성 전략

### 전략 1: 프롬프트 기반 일관성 (기초)

모든 이미지 프롬프트에 동일한 캐릭터 설명 블록을 삽입한다.

**캐릭터 설명 블록 구조:**
```
[CHARACTER_BLOCK_START]
Character: [이름]
Species: [종]
Body shape: [형태 설명]
Color: [색상 설명]
Key features: [식별 특징 3개]
Style: [일러스트 스타일]
Expression: [이 장면의 표정]
Pose: [이 장면의 포즈]
[CHARACTER_BLOCK_END]
```

### 전략 2: 이미지 참조 기반 일관성 (중급)

첫 번째 생성된 캐릭터 이미지를 참조 이미지로 활용한다.

**Midjourney:**
- `--cref [URL]` — 캐릭터 참조 (외모 일관성)
- `--cw 100` — 캐릭터 가중치 최대
- `--sref [URL]` — 스타일 참조 (화풍 일관성)
- `--sw 80` — 스타일 가중치

**Leonardo AI:**
- Character Reference 기능 활성화
- Guidance Scale: 7~8 (너무 높으면 경직)
- Image2Image 강도: 0.3~0.5

### 전략 3: 모델 트레이닝 기반 일관성 (고급)

LoRA (Low-Rank Adaptation) 모델 트레이닝으로 캐릭터를 학습시킨다.

**트레이닝 데이터 준비:**
- 30~40장의 참조 이미지 (다양한 각도, 표정, 포즈)
- 배경은 단색 또는 투명
- 일관된 스타일로 생성
- 캡션: "tori_character, baby triceratops, [포즈/표정 설명]"

**트레이닝 파라미터 권장값:**
- Epochs: 10~20
- Learning Rate: 1e-4
- Batch Size: 1~2
- Resolution: 512×512 또는 768×768

### 일관성 검증 체크리스트

각 삽화 생성 후 다음을 확인:

□ 실루엣이 참조 시트와 일치하는가?
□ 색상 팔레트가 COLOR_DNA와 일치하는가?
□ 식별 특징(뿔, 무늬 등)이 모두 존재하는가?
□ 머리:몸 비율이 설정 범위 내인가?
□ 다른 캐릭터와의 크기 비율이 올바른가?
□ 손가락 수가 정상인가? (AI 생성의 대표적 오류)
□ 좌우 대칭이 필요한 요소가 대칭인가?

하나라도 실패 시 AI 일러스트레이터에 재생성을 요청한다.

## 캐릭터 유형별 설계 가이드

### 동물 캐릭터 (가장 빈도 높음)

- 실제 동물의 대표적 특징 2~3개만 차용하고 나머지는 단순화
- 의인화 수준: 이족 보행 + 표정 + 손 사용 (영아: 사족 보행도 가능)
- 의상: 유아용은 최소한(모자 또는 스카프 1개), 초저용은 풀 의상 가능
- 종류: 포유류 > 파충류 > 조류 > 곤충 순으로 친숙도 높음

### 사람 아이 캐릭터

- 대상 연령과 비슷한 나이로 설정 (또래 동일시 효과)
- 피부색, 머리색은 다양성을 고려하되 교사의 의도를 우선
- 의상으로 성격 표현 (밝은 색=활발, 줄무늬=장난꾸러기)
- 한국적 요소 반영 시: 현대적 의상 기본, 전통 요소는 맥락에 따라

### 사물 의인화 캐릭터

- 원래 사물의 형태를 70% 이상 유지
- 눈과 입만 추가하는 것이 가장 효과적
- 팔다리 추가 시 사물 형태에서 자연스럽게 파생
- 크기 관계: 실제 사물의 상대적 크기를 유지하면 유머 발생

### 판타지/상상 존재

- 텐트 속의 마법 '외로움 괴물' 같은 추상 감정의 시각화
- 기존 동물/사물의 조합이 순수 창작보다 인식이 쉬움
- 귀여움 요소 필수: 크고 둥근 눈, 통통한 몸, 부드러운 질감
- 무서운 존재도 "귀여운 무서움"으로 (영아~유아 대상)
