# /qa — 체계적 QA 테스트

> gstack 패턴 적용: 4가지 모드로 체계적으로 품질을 검증합니다.

## 4가지 QA 모드

사용법: `/qa` (기본: diff-aware) 또는 `/qa full`, `/qa quick`, `/qa regression`

### 모드 1: diff-aware (기본)
최근 변경사항만 집중 테스트합니다.

1. `git diff --name-only HEAD~3`으로 최근 변경 파일 파악
2. 변경된 페이지/API만 대상으로 테스트
3. 변경이 영향을 미치는 연관 기능도 확인

### 모드 2: full
전체 서비스를 처음부터 끝까지 테스트합니다.

### 모드 3: quick
빌드 + 테스트 + 주요 페이지 접근만 확인합니다.

### 모드 4: regression
과거 버그 재발 여부를 집중 확인합니다.

## 테스트 절차

### Phase 1: 빌드 검증
```bash
npm run build    # Edge Runtime 호환 빌드
npm test         # 61개 유닛 테스트
```
실패 시 → 즉시 보고, QA 중단

### Phase 2: 페이지 접근성 (preview MCP 활용)
dev 서버 기동 후 각 페이지 접근:

| 페이지 | 확인 사항 |
|--------|----------|
| `/` (랜딩) | CTA 버튼 동작, 동화 갤러리 스와이프, 후기 표시 |
| `/pricing` | 상품 2종 표시, 가격 정확성, 결제 버튼 |
| `/community` | 동화 목록 로드, 무한 스크롤 |
| `/diy` | 6개 스토리 카드 표시, 선택 → 편집기 진입 |
| `/login` | 카카오/구글 소셜 로그인 버튼 |
| `/library` | (로그인 필요) 내 동화 목록 |

### Phase 3: 핵심 플로우 테스트
1. **동화 생성 플로우**: 랜딩 → 온보딩 → 대화 → 동화 완성 (게스트 5턴)
2. **DIY 플로우**: /diy → 스토리 선택 → 이미지 정렬 → 텍스트 작성
3. **결제 플로우**: /pricing → 상품 선택 → 결제창 호출 (실결제 X)

### Phase 4: 반응형 & 다크모드
preview MCP의 resize로 확인:
- **모바일** (375px): 레이아웃 깨짐 없는지
- **태블릿** (768px): 그리드 전환 정상인지
- **다크모드**: 텍스트 가독성, 배경 전환

### Phase 5: 콘솔 & 네트워크 에러
- `preview_console_logs(level: error)` → 에러 0건 확인
- `preview_network(filter: failed)` → 실패 요청 0건 확인

### Phase 6: 회귀 테스트 (regression 모드)
과거 버그 재발 확인:
- [ ] 이중 엔티티 인코딩 (`&amp;amp;`) 없는지
- [ ] 티켓 우회 (`/?action=start` 시 티켓 체크)
- [ ] 동화 파싱 3가지 전략 모두 동작

## Health Score 계산

| 카테고리 | 가중치 | 감점 기준 |
|----------|--------|----------|
| 빌드 성공 | 20% | 실패 시 0점 |
| 테스트 통과 | 15% | 실패 테스트당 -3% |
| 콘솔 에러 | 15% | 에러 1건당 -5% |
| 네트워크 실패 | 15% | 실패 요청당 -5% |
| 핵심 플로우 | 15% | 플로우 실패당 -15% |
| 반응형 | 10% | 깨진 레이아웃당 -5% |
| 다크모드 | 5% | 가독성 문제당 -2% |
| 회귀 | 5% | 재발 버그당 -5% |

## 주의사항

- QA는 **읽기 전용**입니다 — 코드를 수정하지 않습니다
- 실결제 테스트는 하지 않습니다 (결제창 호출까지만)
- 로그인 필요 페이지는 접근 가능 여부만 확인합니다
- 문제 발견 시 수정은 사용자 승인 후 별도 진행합니다
