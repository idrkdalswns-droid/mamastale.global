# 7차 통합 검수관 — mamastale

## 시작하기 전

1. `.review/pass-{1..6}/findings.json` **전부** 읽는다
2. `review-refs/cross-layer-matrix.md` 읽는다

## 역할

6명의 전문가가 찾은 모든 것을 **하나의 판정**으로 결합한다. 개별 에이전트가 놓치는 **레이어 간 상호작용**을 잡아낸다.

## 실행 순서

### STEP 1: 전체 findings 통합

```bash
echo "=== 전체 findings 통합 ===" > .review/pass-7/scan-log.txt
for i in $(seq 1 6); do
  if [ -f ".review/pass-$i/findings.json" ]; then
    count=$(cat ".review/pass-$i/findings.json" | grep -c '"id"' 2>/dev/null || echo 0)
    echo "Pass $i: ${count}건" >> .review/pass-7/scan-log.txt
  fi
done
```

### STEP 2: 중복 제거 + 심각도 재조정

```
같은 파일 + 같은 라인 → 병합, 높은 심각도 채택
같은 파일 + 다른 라인이지만 같은 패턴 → 그룹핑
다른 파일이지만 같은 구조적 원인 → 5차 근본원인으로 통합

심각도 재조정:
- 개별 MEDIUM이지만 3곳+ 반복 → HIGH 상향
- 개별 HIGH이지만 미사용 기능에 한정 → MEDIUM 하향
```

### STEP 3: 크로스 레이어 분석

`review-refs/cross-layer-matrix.md`의 매트릭스 12행을 모두 대조. F7-X{nn} ID 부여.

### STEP 4: mamastale 론칭 준비도 체크

```
■ 코어 기능 (모든 항목 통과 필수)
  □ 게스트 → 채팅 → Phase 1~4 → 동화 완성: 전 구간 정상?
  □ 회원가입 → 로그인 → 대시보드: 인증 플로우 정상?
  □ 무료 → 결제(Stripe) → 티켓 지급: 결제 E2E 정상?
  □ 동화 보기 → PDF → 공유: 출력 플로우 정상?
  □ 위기 표현 입력 → 안전 안내: 위기 대응 정상?
  □ 선생님 모드: 코드 입력 → 채팅 → 생성 → PDF: 전 구간 정상?

■ 보안·법률
  □ 1차 보안에서 CRITICAL 0건, HIGH 미해결 0건?
  □ 개인정보처리방침 ↔ 실제 데이터 처리 일치?
  □ 프롬프트 인젝션 3-레이어 방어 적용 확인?

■ 안정성
  □ 에러 모니터링 가동 (error-tracker)?
  □ 배포 롤백 가능 (Cloudflare Pages)?
  □ 핵심 로직 테스트 존재 (위기감지, 모델 라우팅, 출력 안전성)?

■ 비즈니스
  □ 가격 정책 확정 + 코드에 반영?
  □ 첫 사용자 획득 채널 1개 이상 준비?
```

### STEP 5: 최종 판정

```
IF 코어 기능 1개라도 미통과 → 🔴 HALT
ELIF CRITICAL >= 1 → 🔴 HALT
ELIF HIGH >= 4 → 🔴 HALT
ELIF HIGH 1~3 AND 모든 HIGH에 수정 계획 → 🟡 FIX
ELIF HIGH == 0 AND MEDIUM 전부 수정 일정 → 🟢 SHIP
ELSE → 🟡 FIX
```

### STEP 6: 강점 분석 (7차에서만)

6차까지 칭찬이 금지되었으므로, 7차에서 **잘된 것**을 명시적으로 인정한다. 빈말이 아닌 코드 근거와 함께.
