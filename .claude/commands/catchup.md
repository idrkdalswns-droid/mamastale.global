# /catchup — 현재 브랜치 변경사항 파악

현재 git 브랜치에서 변경된 파일들을 읽고 컨텍스트를 복원합니다.

## 순서

1. `git status`로 현재 상태 확인
2. `git diff --name-only HEAD~5`로 최근 5개 커밋의 변경 파일 목록 확인
3. 변경된 파일들을 모두 읽어서 현재 작업 맥락 파악
4. `git log --oneline -10`으로 최근 커밋 히스토리 확인
5. 요약: 최근 작업 내용, 변경된 파일, 현재 상태

## 용도

- `/clear` 후 컨텍스트 복원
- 새 세션 시작 시 빠른 캐치업
- 작업 중단 후 재개 시
