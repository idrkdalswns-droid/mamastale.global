# 크로스 레이어 이슈 패턴 매트릭스

7차 통합 검수관이 사용하는 레이어 간 상호작용 패턴.
단독으로는 MEDIUM이지만 조합되면 HIGH/CRITICAL이 되는 패턴을 명시한다.

## 매트릭스

| 결합 | 패턴 | 조합 심각도 | 탐지 방법 |
|------|------|:----------:|----------|
| 보안×백엔드 | 인증 우회 + 민감 API 무방비 | CRITICAL | API 라우트별 인증 체크 맵핑 → 미보호 라우트에 민감 데이터 접근 여부 |
| 보안×프론트 | XSS + 세션 토큰 접근 가능 | CRITICAL | sanitize 누락 지점에서 document.cookie 접근 가능한지 확인 |
| 보안×비즈니스 | 데이터 유출 가능 + 신뢰 기반 서비스 | CRITICAL | 개인 데이터 유출 시 서비스 신뢰 붕괴 여부 판단 |
| 백엔드×프론트 | API 응답 3초+ + 로딩 UI 미흡 | HIGH | 느린 API 목록과 해당 화면의 로딩 상태 교차 |
| 백엔드×UX | 에러 500 + 에러 UI 미흡 | HIGH | API 에러 핸들링 + 프론트 에러 바운더리 교차 |
| 프론트×UX | 접근성 미달 + 핵심 퍼널 | HIGH | WCAG 위반 지점이 핵심 퍼널에 있는지 |
| UX×비즈니스 | 퍼널 마찰 3곳+ + 가격 저항 | HIGH | 무료→유료 전환 경로의 클릭 수 × 이탈률 추정 |
| 아키텍처×비즈니스 | 확장 한계 + 성장 목표 초과 | HIGH | 아키텍처 병목 시점 vs 비즈니스 성장 곡선 교차 |
| 보안×아키텍처 | 시크릿 로테이션 불가 + 단일 키 운영 | MEDIUM→HIGH | 키 유출 시 전체 서비스 영향 범위 |
| 프론트×아키텍처 | 번들 500KB+ + CDN 미사용 | MEDIUM | LCP와 번들 사이즈의 상관관계 |
| 백엔드×아키텍처 | N+1 쿼리 5곳+ + 인덱스 미설정 | MEDIUM→HIGH | 트래픽 증가 시 DB 병목 시점 추정 |
| UX×아키텍처 | 모바일 느림 + Edge Runtime 제약 | MEDIUM | 모바일 성능과 서버 응답 시간 교차 |

## 탐지 프로세스

7차 통합 에이전트는 다음 순서로 크로스 레이어 분석을 수행한다:

1. **1~6차 findings 전수 로드**: `.review/pass-{1..6}/findings.json` 읽기
2. **매트릭스 대조**: 위 표의 각 행에 대해, 해당 패턴의 양쪽 발견이 모두 존재하는지 확인
3. **조합 심각도 상향**: 개별 MEDIUM인데 조합이 HIGH/CRITICAL이면 심각도 상향 판정
4. **결과를 F7-X{nn} ID로 기록**

## 예시

```
1차 보안: [F1-003] MEDIUM — /api/community에 Rate Limit 없음
4차 UX: [F4-005] LOW — 커뮤니티 페이지 로딩이 약간 느림

7차 교차 분석:
[F7-X01] HIGH — F1-003 × F4-005 결합
Rate Limit 없는 /api/community를 공격자가 초당 1000회 호출하면
DB 과부하 → 커뮤니티 뿐 아니라 전체 서비스 응답 지연
개별적으로는 MEDIUM+LOW이지만 DDoS 벡터로서 HIGH
```
