# UX 휴리스틱 체크리스트 — mamastale 전용

## Nielsen 10 휴리스틱 평가 기준

0점: 위반 없음
1점: 표면적 문제 — 다음 릴리즈에서 수정
2점: 사소한 사용성 문제 — 우선순위 낮음
3점: 주요 사용성 문제 — 수정 중요, HIGH
4점: 사용성 재앙 — 반드시 수정, CRITICAL

### H1: 시스템 상태 가시성
```
4점: 현재 어디 있는지, 뭘 하는 중인지 전혀 알 수 없음
3점: Phase 표시는 있으나 진행률/남은 시간 부재
2점: 로딩 중 피드백이 3초 이상 없음
1점: 일부 비동기 작업에 로딩 표시 누락
0점: 모든 상태가 즉시 가시적

mamastale 평가 대상:
- 4단계 Phase 표시기의 명확성 (엄마 모드)
- 5단계 Phase 표시기 (선생님 모드)
- AI 응답 대기 중 타이핑 인디케이터
- Stripe 결제 처리 중 상태 표시
- 동화 생성 중 진행률 (Phase 4)
- PDF 다운로드 중 상태 (선생님 모드)
```

### H5: 오류 방지
```
4점: 결제/삭제 등 돌이킬 수 없는 행동에 확인 없음
3점: 필수 입력 검증이 제출 후에만 발생 (실시간 검증 없음)
2점: 오류 발생 후 입력 데이터가 사라짐
1점: 일부 엣지 케이스에서 의도치 않은 행동
0점: 모든 위험 행동에 사전 방지책

mamastale 평가 대상:
- Stripe 결제 전 확인 대화상자
- 동화 삭제 전 확인
- 채팅 중 브라우저 닫기 시 경고
- 선생님 모드: 온보딩 뒤로가기 시 데이터 보존
- 선생님 모드: 타이머 만료 시 자동 생성 전 확인
```

### H9: 에러 인식·진단·복구
```
4점: 에러 시 기술적 메시지만 표시, 복구 방법 없음
3점: 에러 메시지가 있으나 해결책 안내 없음
2점: 에러 복구 경로가 있으나 눈에 띄지 않음
1점: 일부 에러에 "다시 시도" 누락
0점: 모든 에러에 원인+해결책+복구 버튼

mamastale 평가 대상:
- AI 응답 실패 시 "다시 시도" 버튼 (isError 상태)
- Stripe 결제 실패 시 안내
- 네트워크 끊김 시 채팅 복구
- 선생님 모드: 코드 입력 실패 시 안내
- 위기 상황 감지 후 안전 안내 즉시 표시
```

## 페르소나 시뮬레이션 (mamastale 3명)

**페르소나 A — 김지은 (30세, 산후우울증, 첫째 6개월)**
```
상태: 극도로 지침, 아기가 자는 틈 15분, 손이 떨림
디바이스: iPhone SE (작은 화면), 한 손 조작
시뮬레이션:
1. 랜딩 도착 → 3초 내 "이게 뭔지" 이해 가능한가?
2. CTA 터치 → 엄지로 닿는 위치인가? (하단 44px+?)
3. 온보딩 → 지쳐서 읽기 싫을 때 스킵 가능한가?
4. 채팅 시작 → 첫 질문이 위압적이지 않은가?
5. 울면서 타이핑 → 오타가 많을 때 AI가 이해하는가?
6. 위기 표현 → "죽고 싶다" 입력 시 적절한 안내가 즉시 나오는가?
7. 중간 이탈(아기 울음) → 30분 후 돌아왔을 때 대화가 유지되는가?
8. 동화 완성 → 감정적으로 격해진 상태에서 그라운딩 안내가 있는가?
```

**페르소나 B — 박서현 (38세, 양육 번아웃, 둘째 5세)**
```
상태: 화가 많음, "나만의 시간" 30분, 빠르게 결과를 원함
시뮬레이션:
1. "15분 안에 끝나?"라는 의문 → 예상 소요시간 표시가 있는가?
2. Phase 전환이 느리면 → "빨리 넘어가자" 할 수 있는가?
3. 결제 요청 → "지금?" 하는 타이밍이 치유 경험을 방해하지 않는가?
4. 5세 아이 기준 동화 → 연령 선택이 결과에 반영되는 게 체감되는가?
5. 재방문 → "지난번 이야기 이어서"가 가능한가?
```

**페르소나 C — 이다연 (35세, 경력단절, 첫째 3세)**
```
상태: 분석적, 다른 서비스와 비교, 가격에 민감
시뮬레이션:
1. 가격 정보 → 바로 찾을 수 있는가? 숨겨져 있으면 불신
2. 다른 사용자 후기 → 진짜 사용자인지 판단할 근거가 있는가?
3. 무료 체험 범위 → 무료로 뭘 할 수 있는지 명확한가?
4. 동화 품질 샘플 → 결제 전에 결과물을 볼 수 있는가?
5. 데이터 보안 → "내 상담 내용이 안전한가?" 안심할 근거가 있는가?
```

## 감정 디자인 체크리스트 (치유 서비스 특화)

```
□ 색상 톤: 채팅 화면이 차분하고 안전한 느낌인가? (높은 채도 = 불안)
□ 마이크로카피: 버튼 텍스트가 부드러운가? ("제출" vs "마음을 나눠주세요")
□ 빈 상태: 처음 방문 시 "환영"의 느낌이 있는가?
□ Phase 전환: 갑작스럽지 않고 자연스러운가?
□ 에러 톤: 기술적이 아니라 인간적인가? ("500 Internal Error" vs "잠시 문제가 생겼어요")
□ 축하 순간: 동화 완성 시 축하가 과하지 않고 진심 어린가?
□ 결제 요청 톤: 치유 경험의 연장인가 상업적 전환인가?
□ 위기 안내 톤: 당황시키지 않으면서 진지한가?
□ 선생님 모드: 교사 전문성을 존중하는 톤인가?
```

## mamastale 핵심 플로우 (검수 대상)

```
1. 첫 방문 → 온보딩 → 채팅 시작 (게스트)
2. 채팅 Phase 1~4 → 동화 완성 → 동화 보기
3. 회원가입 → 로그인 → 대시보드
4. 무료 → 결제(Stripe) → 유료 전환
5. 동화 공유 → 커뮤니티 열람
6. 오류 발생 → 복구 경로
7. 위기 상황 → 안전 안내
8. 선생님 모드: 코드 입력 → 온보딩 → 채팅 → 미리보기 → PDF
```
