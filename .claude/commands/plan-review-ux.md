# 4차 UX 감사관

plan.md를 읽고 **"이 계획대로 빌딩하면 사용자가 어떤 경험을 하게 되는가?"**를 시뮬레이션한다.

## 시작하기 전

반드시 `plan-review-refs/ux-anti-patterns.md` 를 먼저 읽어라. 플랜에서 흔히 빠지는 UX 요소 5가지와 페르소나 체크포인트가 정리되어 있다.

## 검토 관점

### 사용자 여정 영향 분석
```
□ 이 기능이 기존 사용자 플로우의 어디에 삽입되는가?
  - 새 화면이 추가되는가? → 기존 플로우의 클릭 수가 늘어나지 않는가?
  - 기존 화면을 수정하는가? → 기존 사용자가 혼란스럽지 않은가?
□ 첫 사용자(처음 보는 사람)가 이 기능을 이해하는 데 몇 초 걸리는가?
  - 3초 규칙: 3초 내 다음 행동을 알 수 없으면 이탈
□ 이 기능으로 인해 핵심 퍼널(채팅→동화→결제)에 마찰이 추가되는가?
```

### 페르소나 시뮬레이션

계획의 영향을 3명의 페르소나로 시뮬레이션:

**지은 (산후우울증, 30세)**: 집중력 극히 낮음, 한 손 조작, 감정적으로 불안정
- 이 기능이 지은에게 추가 인지 부하를 주는가?
- 지은이 이 기능을 만났을 때 안전하다고 느끼는가?

**서현 (양육 번아웃, 38세)**: 시간 부족, 빠른 결과 지향
- 이 기능이 서현의 소요 시간을 늘리는가?
- 서현이 이 기능을 스킵할 수 있는가?

**다연 (경력단절, 35세)**: 분석적, 가격 민감, 신뢰 중시
- 이 기능이 서비스의 신뢰감을 높이는가 낮추는가?
- 이 기능이 결제 전환에 기여하는가 방해하는가?

### 감정 디자인 (치유 서비스 특화)
```
□ 이 기능이 사용자의 감정적 안전감을 해치지 않는가?
□ 치유적 경험의 흐름을 방해하지 않는가?
  - 예: 채팅 중 갑작스러운 팝업, 결제 요청, 설문 등
□ 에러 발생 시 사용자가 받는 감정적 충격을 고려했는가?
□ 위기 상황(자살 표현)에서 이 기능이 안전하게 작동하는가?
```

### 인지 부하
```
□ 한 화면에 사용자가 처리해야 할 정보량이 적절한가?
  - 선택지가 3개 이하인가?
  - 필수 입력이 3개 이하인가?
□ 새 개념이나 용어를 사용자에게 학습시키지 않는가?
□ 사용자가 "이전에 뭘 했는지" 기억해야 하는 구조가 아닌가?
```

### 1~3차 교차 분석
```
보안(1차): 인증 요구 → 사용자에게 로그인 강제가 UX를 해치는 시점은?
백엔드(2차): API 지연 → 사용자가 대기하는 동안의 경험 설계가 있는가?
프론트(3차): 로딩 상태 → 구체적으로 어떤 마이크로카피를 보여줄 건가?
프론트(3차): 접근성 → 취약 사용자(산후우울증)에게 특히 치명적인 접근성 이슈는?
```
