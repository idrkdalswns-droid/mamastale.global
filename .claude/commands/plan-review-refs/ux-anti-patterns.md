# 플랜에서 흔히 빠지는 UX 요소

## 패턴 1: 사용자 여정 단절
```
plan.md: "동화 공유 기능을 추가한다"
빠진 것:
  - 공유 링크를 받은 사람(비로그인)의 경험은?
  - 공유된 동화를 본 후 "나도 만들어보고 싶다" → 다음 행동이 설계되어 있는가?
  - 공유한 사람에게 "3명이 당신의 동화를 읽었어요" 피드백이 있는가?
질문: "이 기능의 시작점부터 끝점까지 사용자 여정을 플로우차트로 그릴 수 있는가?"
```

## 패턴 2: 감정 흐름 무시 (치유 서비스 특수)
```
plan.md: "채팅 중간에 추천 기능을 보여준다"
빠진 것:
  - Phase 1(공감)에서 울고 있는 사용자에게 갑자기 추천?
  - Phase 3(은유)에서 몰입 중인데 팝업?
  - 감정적으로 취약한 순간에 상업적 요소가 노출되면?
질문: "이 기능이 사용자의 감정 흐름을 방해하는 시점은 없는가?"
```

## 패턴 3: 기존 플로우에 마찰 추가
```
plan.md: "온보딩에 연령 선택 외에 관심사 선택도 추가한다"
빠진 것:
  - 현재 온보딩 클릭 수: 7 (이미 많음)
  - 관심사 선택 추가 시: 7+2 = 9 → 이탈률 증가
  - 이 정보를 실제로 활용하는가? 수집만 하고 안 쓰면 낭비
  - 스킵 가능한가?
질문: "핵심 퍼널(첫 방문 → 첫 동화)의 클릭 수가 늘어나는가? 늘어난다면 그만큼의 가치가 있는가?"
```

## 패턴 4: 에러 시 감정적 충격 미고려
```
plan.md: "결제 실패 시 에러 메시지를 표시한다"
빠진 것:
  - 에러 메시지 톤: "결제 실패" vs "결제가 완료되지 않았어요. 다시 시도해볼까요?"
  - 동화 생성 실패 시: Phase 4까지 20분 대화한 뒤 실패하면 감정적 충격 극대
  - 복구 경로: 에러 후 사용자가 뭘 해야 하는지 명확한가?
질문: "에러가 발생하는 모든 지점에서 사용자가 받는 감정적 충격을 고려했는가?"
```

## 패턴 5: 재방문 시나리오 부재
```
plan.md: "새 기능 X를 추가한다"
빠진 것:
  - 첫 사용과 재사용의 경험이 다른가?
  - 2주 만에 돌아온 사용자가 "이전에 뭘 했는지" 볼 수 있는가?
  - 미완성 상태(채팅 중단)에서 돌아왔을 때 이어서 할 수 있는가?
질문: "이 기능은 한 번 쓰고 끝인가, 반복 사용되는가? 반복이라면 재방문 시 경험을 설계했는가?"
```

## 페르소나 체크포인트 (모든 plan에 적용)

```
모든 plan.md에 대해 3명의 페르소나로 5초 시뮬레이션:

지은(산후우울증): "지금 극도로 지쳐있는데, 이 기능이 나를 더 피곤하게 만드는가?"
서현(번아웃): "15분밖에 없는데, 이 기능이 내 시간을 빼앗는가?"
다연(경력단절): "이 기능이 서비스 신뢰를 높이는가 낮추는가?"

하나라도 "이 기능 때문에 더 나빠진다"면 → 🟠 HIGH
```
