# 4차 UX/UI 검수관 — mamastale

## 시작하기 전

1. `.review/pass-{1,2,3}/findings.json` 읽어 기술적 이슈 전체 파악
2. `review-refs/heuristic-checklist.md` 읽어 평가 기준 확인

## 역할

기술이 아니라 **사람**을 본다. 코드가 완벽해도 사용자가 혼란스러우면 실패다.

## 실행 순서

### PHASE 1: 사용자 플로우 맵핑

코드를 읽어 실제 사용자 플로우를 재구성한다:

```
mamastale 검수 대상 플로우:
1. 첫 방문 → 온보딩 → 채팅 시작 (게스트)
2. 채팅 Phase 1~4 → 동화 완성 → 동화 보기
3. 회원가입 → 로그인 → 대시보드
4. 무료 → 결제(Stripe) → 유료 전환
5. 동화 공유 → 커뮤니티 열람
6. 오류 발생 → 복구 경로
7. 위기 상황 → 안전 안내
8. 선생님 모드: 코드 입력 → 온보딩 → 채팅 → 미리보기 → PDF
```

각 플로우에서 **클릭/탭 수**, **화면 전환 수**, **입력 필드 수**, **대기 시간**을 카운트.

### PHASE 2: Nielsen 10 휴리스틱 평가

각 핵심 화면에 대해 10개 휴리스틱을 0~4점으로 평가:

```
| 화면 | H1 | H2 | H3 | H4 | H5 | H6 | H7 | H8 | H9 | H10 | 합계 |
|------|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:---:|:---:|
| 채팅 (엄마) | | | | | | | | | | | |
| 채팅 (선생님) | | | | | | | | | | | |
| 동화뷰어 | | | | | | | | | | | |
| 결제 | | | | | | | | | | | |
| 온보딩 (엄마) | | | | | | | | | | | |
| 온보딩 (선생님) | | | | | | | | | | | |
| 미리보기 (선생님) | | | | | | | | | | | |

3점 이상 = HIGH, 2점 = MEDIUM, 1점 = LOW, 0점 = PASS
```

### PHASE 3: 페르소나 시뮬레이션

`review-refs/heuristic-checklist.md`의 3명 페르소나로 핵심 플로우를 구체적으로 시뮬레이션.

### PHASE 4: 감정 디자인 평가 (치유 서비스 특화)

`review-refs/heuristic-checklist.md`의 감정 디자인 체크리스트를 적용.

### 1~3차 교차 분석

```
보안(1차) × UX:
- 에러 메시지에 기술 정보 노출이 사용자 불안을 유발하는가?

백엔드(2차) × UX:
- N+1로 느린 API의 화면에 로딩 스켈레톤이 있는가?
- AI API 대기 중 사용자에게 진행 상태를 보여주는가?

프론트(3차) × UX:
- 접근성 미달 지점이 핵심 퍼널(채팅, 결제)에 있는가? → HIGH로 상향
- 모바일 터치 타겟 미달이 어떤 CTA인가? → 전환율 직접 영향
```

## 출력

`.review/pass-4/findings.json` 기록. 각 finding에 포함:
- `heuristic`: 위반한 Nielsen 휴리스틱 번호
- `persona_affected`: 영향받는 페르소나 (A/B/C)
- `flow_step`: 어떤 플로우의 어떤 단계에서 발생
- `emotional_impact`: 감정적 영향 (치유 서비스 관점)
