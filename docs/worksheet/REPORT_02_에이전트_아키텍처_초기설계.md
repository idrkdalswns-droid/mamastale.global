# 마마스테일 AI 활동지 에이전트 아키텍처 설계 리서치

마마스테일(MAMASTALE)의 치유 동화 기반 활동지 자동 생성 서비스는 **Claude Haiku 4.5를 주력 모델로, 결정론적 HTML/SVG 템플릿에 AI 콘텐츠를 주입하는 하이브리드 파이프라인 아키텍처**가 최적이다. 이미지 생성 모델(DALL-E, Midjourney) 없이 프로그래밍 방식의 SVG + 오픈소스 클립아트만으로 활동지 시각 품질을 확보할 수 있으며, 월 10,000매 기준 **API 비용 $21~42 수준**으로 운영 가능하다. 2019 개정 누리과정은 2025~2026년에도 그대로 유지되며, 만 3~5세 통합 59개 내용항목을 기반으로 연령별 3~4종 활동지를 차등 생성하는 것이 핵심 차별화 포인트다.

---

## 누리과정 5개 영역과 59개 내용항목 전수 체계

2019 개정 누리과정(교육부 고시 제2019-189호)은 2020년 3월부터 시행되어 **2025~2026년 현재까지 변경 없이 유지**되고 있다. 2024년 개정은 0~2세 표준보육과정에만 적용되었고, 3~5세 누리과정의 다음 개정은 2029년경으로 예상된다. 핵심 철학은 **유아중심·놀이중심**이며, 기존 연령별 369개 항목을 연령 통합 **59개 내용항목**으로 대폭 간소화했다.

**5개 영역별 핵심 내용범주와 내용**은 다음과 같다.

**신체운동·건강** 영역은 신체활동 즐기기(신체 인식·움직임 조절·기초 운동·자발적 참여), 건강하게 생활하기(위생·식습관·휴식·질병예방), 안전하게 생활하기(안전놀이·미디어사용·교통안전·재난대처)의 **3개 내용범주, 12개 내용**으로 구성된다. **의사소통** 영역은 듣기와 말하기(관심 있게 듣기·경험 말하기·적절한 단어 사용·관련 말하기·바른 태도·고운 말), 읽기와 쓰기에 관심 가지기(말과 글 관계·읽기 관심·글자 형태 표현), 책과 이야기 즐기기(상상하기·말의 재미·이야기 짓기)의 **3개 내용범주, 12개 내용**이다. **사회관계** 영역은 나를 알고 존중하기(자기 인식·감정 표현·자립), 더불어 생활하기(가족·친구·갈등해결·존중·예의·약속규칙), 사회에 관심 가지기(지역사회·우리나라·다양한 문화)의 **3개 내용범주, 12개 내용**이다. **예술경험** 영역은 아름다움 찾아보기(자연·예술적 요소), 창의적으로 표현하기(노래·리듬·움직임·미술·극놀이), 예술 감상하기(감상·존중·전통예술)의 **3개 내용범주, 10개 내용**이다. **자연탐구** 영역은 탐구과정 즐기기(호기심·참여·다른 생각 관심), 생활 속에서 탐구하기(물체 특성·수량·위치방향모양·비교·분류·도구기계), 자연과 더불어 살기(동식물·생명존중·날씨계절)의 **3개 내용범주, 13개 내용**이다.

마마스테일에서 이 체계가 갖는 의미는 명확하다. 누리과정은 연령별 구분을 폐지했지만, **실제 현장에서는 연령별 발달 차이에 따른 활동 차등화가 필수**적이다. 따라서 마마스테일이 동일한 59개 내용항목을 기준으로 만3/4/5세별 난이도를 자동 조절하는 것 자체가 시장 차별화 요소가 된다. 모든 생성 활동지에 해당 누리과정 영역·내용범주 태그를 부여하면, 교사들의 **보육일지 작성과 어린이집 평가제 대응**을 직접 지원할 수 있다.

**2024 개정 어린이집 평가제**(2024.7.3 시행)는 등급제를 폐지하고 서술형 평가로 전환했으며, '영유아 중심 일과 운영', '영유아가 주도하는 배움 지원' 등 **6개 평가영역**을 통해 놀이중심·과정중심 교육을 강조한다. 활동지는 학습지 형태가 아닌 **놀이 확장 자료**로 설계되어야 하며, 관찰 기록을 지원하는 필드(교사 관찰 메모란 등)를 포함하면 평가제 부합도를 높일 수 있다.

## 연령별 발달 수준에 맞춘 최적 활동지 3~4종 선정

### 만3세: 큰 그림, 단순 반복, 감각 중심

만3세는 소근육이 미성숙하여 굵은 크레용을 쥐는 수준이며, **주의집중 시간이 6~9분**에 불과하다. 어휘는 약 900~1,000단어, 3~4어절 문장을 사용한다. 이 연령에 최적화된 활동지 4종은 다음과 같다.

**①색칠하기(Coloring)** — 동화 캐릭터의 대형 윤곽선(2~3mm)에 색을 칠하는 활동으로, 페이지당 색칠 영역 3~5개 이내. 색상 인식 발달(Jalil et al., 2013: 0~8세 시각 자극으로서 색상이 중요)과 손 근력 강화에 효과적이다. 한국 유아교육 플랫폼에서 **가장 보편적으로 사용되는 활동지 유형 1위**다. **②스티커·짝짓기(Sticker/Matching)** — 지정된 위치에 스티커를 붙이거나 동일한 그림을 연결하는 활동. 집게 잡기(pincer grasp) 발달과 시각 변별력을 키우며, 연필 잡기가 어려운 연령에 접근성이 높다. **③선긋기(Line Tracing)** — 두 이미지를 연결하는 직선·곡선 따라 그리기. 경로 폭 5mm 이상, 점선 가이드 제공. 쓰기 준비 기술의 기초를 형성한다. **④같은 그림 찾기(Visual Matching)** — 3~4개 보기 중 동화 캐릭터와 같은 그림 또는 그림자 맞추기. 시각 변별과 인지 범주화 능력을 발달시킨다.

### 만4세: 문제해결, 숫자 인식, 가위 활동 도입

만4세는 가위로 직선을 자를 수 있고, 한 발로 서기가 가능하며, **주의집중 시간 8~12분**이다. 어휘 1,500~2,000단어, "왜?"라는 질문을 반복하며, 글자 인식이 시작된다. 최적 활동지 4종은 다음과 같다.

**①미로찾기(Maze)** — 동화 주제 미로("공주가 마법 숲을 통과하도록 도와줘"). 1~2개 분기점, 넓은 통로. 문제해결력, 소근육, 쓰기 준비, 지속적 주의력을 동시에 발달시킨다(Early Childhood Research Quarterly). **②점잇기(Connect-the-Dots)** — 숫자 1~10(또는 15)을 연결해 동화 캐릭터를 완성. 수 인식 + 순서 이해 + 소근육 조절을 하나의 활동에 통합한다. **③글자·숫자 따라쓰기(Tracing)** — 한글 자음(ㄱ, ㄴ, ㄷ) 또는 숫자(1~5)를 점선 위에 따라 쓰기. 한국 교육 사이트(참잘했어요, 애니스쿨, 니즈폼)에서 이 연령대 핵심 활동으로 제공된다. **④오리기·붙이기(Cut & Paste)** — 동화 캐릭터 조각을 오려 장면을 구성. 양손 협응과 계획 능력을 발달시키며, 꼬망세·키드키즈에서 만들기/크래프트 활동으로 매우 인기가 높다.

### 만5세: 논리, 쓰기, 창의적 표현

만5세는 가위 사용이 숙련되고, 자기 이름과 일부 글자를 쓸 수 있으며, **주의집중 시간 10~15분**이다. 20 이상 세기, 패턴 인식, 간단한 덧뺄셈이 가능하다. 초등학교 이음교육(transition) 관점도 중요하다. 최적 활동지 4종은 다음과 같다.

**①이야기 순서 맞추기(Story Sequencing)** — 4~6개 장면을 시간 순서로 배열하거나 번호 매기기. 논리적 순서 파악은 핵심 학업 준비 기술이며, 동화 콘텐츠를 직접 활용하는 마마스테일 최적 활동 유형이다. **②한글 쓰기 연습(Writing Practice)** — 동화 속 핵심 어휘를 따라 쓰고 독립적으로 쓰기. 따라쓰기 → 보고쓰기 → 자유쓰기 단계 진행. 글자 교육과 독해력을 동시에 연결한다. **③수학 개념 활동(Math Activities)** — 동화 속 사물 세기, 간단한 덧셈, 숫자 색칠(Color-by-Number). 참쌤스쿨에서 수준별 숫자 색칠놀이를 제공할 정도로 인기가 높다. **④창의적 그리기·독후활동(Creative Drawing/Post-Reading)** — "다음 이야기를 그려보세요", "좋아하는 장면을 그리고 한 문장으로 써보세요" 등. 찬진교육 유아스토리에서 **독후활동지**를 별도 주요 카테고리로 분류할 정도로, 한국 유아교육에서 확립된 활동 유형이다.

### 혼합반 대응: "하나의 동화, 세 가지 수준" 전략

혼합연령반은 한국 유치원의 약 30%를 차지한다. 마마스테일의 전략은 **동일한 동화를 듣되, 색상 코딩된 3단계 활동지**(⭐ 쉬운/⭐⭐ 보통/⭐⭐⭐ 어려운)를 제공하는 것이다. Level 1은 색칠+짝짓기, Level 2는 미로+점잇기+따라쓰기, Level 3은 순서 맞추기+글쓰기+창의 그리기로 구성한다. 활동지 상단 모서리에 난이도 아이콘을 표시하면 교사가 즉시 분류·배포할 수 있다.

## 파이프라인 기반 AI 에이전트 아키텍처 설계

### 왜 파이프라인인가: Supervisor보다 안정적이고 디버깅 가능

Anthropic 공식 가이드("Building Effective Agents")는 "가능한 한 단순하게 시작하라"고 권고한다. 마마스테일의 워크플로우는 **고정된 순서**(동화 분석 → 누리과정 매핑 → 활동지 설계 → PDF 생성 → QA 검수)를 따르므로, Supervisor 패턴보다 **파이프라인(Sequential Pipeline) 패턴**이 최적이다. LangGraph 벤치마크에서도 Supervisor의 "전화놀이 문제"(중간 번역 단계에서 정보 손실)로 인해 고정 워크플로우에서는 파이프라인이 **67% 더 적은 토큰**을 소비하는 것으로 나타났다.

프레임워크(LangChain/LangGraph) 없이 **순수 Python async + Pydantic 상태 모델**로 구현하는 것이 MVP에 가장 적합하다. 각 "에이전트"는 고유한 시스템 프롬프트와 도구를 가진 독립 함수이며, Pydantic 모델을 통해 단계 간 상태를 전달한다.

### 5단계 파이프라인 상세 설계

```
동화 텍스트 + 대상 연령(3/4/5세)
        │
        ▼
[Agent 1] 동화 분석기 (Story Analyzer)
 - Claude Haiku 4.5, 구조화 JSON 출력
 - 핵심 감정 테마, 치유 메시지, 핵심 어휘, 캐릭터, 문화 요소 추출
 - 출력: StoryAnalysis (Pydantic 모델)
        │
        ▼
[Agent 2] 누리과정 매퍼 (Curriculum Mapper)
 - Claude Haiku 4.5
 - 동화 분석 결과를 5개 영역에 매핑
 - 연령별 최적 활동지 3~4종 선정 (사전 정의된 활동 유형 풀에서 선택)
 - 출력: CurriculumMap (선정된 활동 목록 + 누리과정 태그)
        │
        ▼
[Agent 3] 활동지 설계사 (Activity Designer) — 연령별 병렬 실행 가능
 - Claude Haiku 4.5, 활동 유형별 구조화 JSON 스키마
 - 제목(한글), 지시문(한글), 어휘 단어, 난이도 파라미터, 콘텐츠 데이터 생성
 - SVG 일러스트레이션은 Claude가 직접 생성하지 않음 → 알고리즘 생성기/클립아트 라이브러리에서 선택
 - 출력: List[ActivitySpec]
        │
        ▼
[모듈] PDF 생성기 (PDF Generator) — 결정론적, LLM 미사용
 - Jinja2 템플릿 + WeasyPrint
 - SVG 활동 요소(미로, 점잇기 등)는 Python 알고리즘으로 생성
 - 한글 폰트 번들링 (학교안심 둥근미소 + Noto Sans KR)
 - 출력: PDF bytes
        │
        ▼
[Agent 4] QA 검수자 (Quality Reviewer)
 - Claude Haiku 4.5
 - 연령 적합성, 누리과정 정합성, 안전성, 문화 적합성 검증
 - 승인 또는 수정 피드백 → 수정 시 Agent 3으로 루프백 (최대 2회)
 - 출력: QAResult {approved: bool, feedback: str}
```

### 핵심 구현 패턴: 구조화 출력으로 안정성 확보

Claude API의 **구조화 JSON 출력(Structured Outputs)** 기능은 `output_config.format`에 `json_schema`를 지정하면 문법 제약 토큰 생성(constrained decoding)을 통해 **100% 유효한 JSON**을 보장한다. Haiku 4.5에서 지원이 확인되었다. 각 에이전트의 출력을 JSON 스키마로 강제하면 파싱 실패 재시도 로직이 불필요해진다.

```python
output_config={
    "format": {
        "type": "json_schema",
        "json_schema": {
            "name": "activity_sheet",
            "schema": {
                "type": "object",
                "properties": {
                    "title_ko": {"type": "string"},
                    "instructions_ko": {"type": "string"},
                    "activity_type": {"type": "string", 
                        "enum": ["coloring", "matching", "tracing", "maze",
                                 "dot_to_dot", "cutting", "sequencing",
                                 "writing", "math", "creative_drawing"]},
                    "difficulty": {"type": "integer", "minimum": 1, "maximum": 3},
                    "vocabulary_words": {"type": "array", "items": {"type": "string"}}
                },
                "required": ["title_ko", "instructions_ko", "activity_type", "difficulty"]
            }
        }
    }
}
```

### 페르소나 스위칭 vs 개별 에이전트 호출

Vanderbilt 대학 연구에 따르면 "You are an X" 형태의 **직접 역할 지정**이 "Imagine you are..." 형태보다 출력 안정성이 높고, 여러 페르소나를 동시에 부여하면 모델의 내부 신호가 혼란해진다. 마마스테일에서는 각 파이프라인 단계별로 **단일 전문가 페르소나**를 시스템 프롬프트에 할당하는 것이 최적이다. 예를 들어 동화 분석기는 "당신은 20년 경력의 한국 치유 동화 전문가 김 박사입니다", 누리과정 매퍼는 "당신은 누리과정 전문 교사 박선생입니다"와 같이 구체적이고 상세한 페르소나를 부여한다.

## 비용 $21/월로 10,000매 생성하는 최적화 전략

### Claude API 모델별 비용 비교

### 4중 비용 절감 전략

**프롬프트 캐싱**은 시스템 프롬프트(~1,500 토큰)를 5분 TTL로 캐싱하여 입력 비용을 **최대 90% 절감**한다. 캐시 읽기는 기본 가격의 0.1배($0.10/MTok vs $1.00/MTok). **배치 API**는 24시간 처리 대신 **50% 할인**을 제공하며, 주간 활동지 사전 생성에 적합하다. **애플리케이션 레벨 캐싱**은 `hash(story_id + activity_type + age + difficulty)`를 키로 Redis에 결과를 저장하며, 인기 동화에 대해 30~50% 캐시 적중률을 기대할 수 있다. **프롬프트 최적화**로는 JSON 구조화 출력(산문 대비 ~15% 토큰 절감), 동화 전문 대신 요약본 전달, `max_tokens` 1024 제한 등이 있다.

이 모든 최적화를 적용하면 월 10,000매 기준 **$21~42 수준**으로 운영할 수 있다.

## 이미지 생성 모델 없이 고품질 활동지를 만드는 기술 스택

### 4계층 시각 자산 아키텍처

마마스테일의 핵심 설계 원칙은 **유료 이미지 생성 모델 의존도 제로**다. 이를 4계층으로 달성한다.

**1계층: 알고리즘 SVG 생성기** — Python 라이브러리로 미로(labyrinth 패키지, DFS/Kruskal/Prim 알고리즘), 점잇기(svgpathtools로 SVG 경로를 따라 번호 매긴 점 배치), 패턴 완성(SVG 도형 시퀀스), 글자 따라쓰기(stroke-dasharray 속성의 점선 경로) 등을 **결정론적으로** 생성한다. 난이도는 파라미터로 조절된다(만3세: 5×5 격자, 만5세: 8×8 격자).

**2계층: 오픈소스 SVG 클립아트 라이브러리** — OpenClipart(CC0, SVG), SVGRepo(680K+ 벡터), CocoMaterial(손그림 스타일, 오픈소스), unDraw(무료 상업 사용)에서 동물, 자연, 가족, 감정 등 테마별 **200여 개 SVG를 큐레이션**한다. 모두 상업적 사용 무료다.

**3계층: HTML/CSS 순수 레이아웃 요소** — 줄 긋기 영역(CSS 보더), 체크박스, 매칭 테이블, 색칠 블록, 격자 레이아웃은 이미지 없이 CSS만으로 구현한다.

**4계층: Claude SVG 생성(보조)** — Claude에게 간단한 라인아트 SVG 코드를 직접 생성하도록 요청할 수 있으나, 이는 보조 수단으로만 사용하고 핵심 의존은 1~3계층에 둔다.

WeasyPrint는 SVG를 래스터화하지 않고 **PDF 내에서 벡터로 렌더링**하므로, 어떤 DPI에서든 선명한 인쇄 품질을 보장한다. 이미지 생성 비용은 **$0**이다.

### WeasyPrint 한글 렌더링 주의사항과 해결책

WeasyPrint의 CJK 폰트 처리에는 알려진 이슈가 있다. 한글 폰트 사용 시 라틴 텍스트 대비 **약 6배 느려지며**(GitHub issue #2120), `local()` 대신 반드시 `url()`로 폰트를 지정해야 하고(issue #1337), 글리프 누락 시 크래시가 발생한다(issue #1386). **해결책**: Noto Sans KR OTF 파일을 프로젝트에 번들링, `@font-face { src: url('/fonts/NotoSansKR-Regular.otf') }` 사용, WeasyPrint v63+ 사용(CJK 성능 개선), 인스턴스 사전 워밍. **폴백**: WeasyPrint 실패 시 Playwright(Python Puppeteer, headless Chromium)로 해당 문서만 대체 렌더링한다.

| 비교 항목 | WeasyPrint | Playwright(Puppeteer) | wkhtmltopdf |
|-----------|-----------|----------------------|-------------|
| 언어 | Python 네이티브 | Node.js/Python | C++ 바이너리(deprecated) |
| SVG 벡터 출력 | ✅ | ✅ | ⚠️ 제한적 |
| 한글 폰트 | ⚠️ 설정 필요, 느림 | ✅ 우수 | ⚠️ 불안정 |
| CSS @page 지원 | 우수 | 양호 | 미흡 |
| 메모리 사용 | ~50-100MB | ~300MB+ | ~중간 |
| 유지보수 | 활발 | 활발(Google) | **2023.1 아카이브** |

## 결정론적 템플릿 + AI 주입의 하이브리드 안정성 설계

### 템플릿이 레이아웃을, AI가 콘텐츠를 담당하는 원칙

완전 AI 생성 활동지는 실행마다 결과가 달라지고 디버깅이 어려우며, 한글 텍스트 정확도 리스크가 있다. 순수 템플릿은 창의성이 없다. **하이브리드 아키텍처**에서는 사전 검증된 HTML/CSS 템플릿이 페이지 레이아웃, 폰트, 색상, SVG 영역, 브랜딩을 고정하고, AI는 **제목 텍스트, 지시문, 어휘 단어, 난이도 파라미터, 콘텐츠 데이터 포인트만** 생성한다. AI가 HTML/CSS/SVG 코드를 직접 생성하지 않는 것이 핵심 안정성 원칙이다.

필요한 기본 템플릿은 약 **10~15종**이다: 색칠 페이지(`coloring_page.html`), 한글 따라쓰기(`hangul_tracing.html`), 미로(`maze_activity.html`), 매칭 게임(`matching_game.html`), 감정 표현(`emotion_faces.html`), 수세기(`counting_activity.html`), 점잇기(`dot_to_dot.html`), 다른 그림 찾기(`find_difference.html`), 순서 맞추기(`sequence_ordering.html`), 자유 그리기(`free_drawing.html`), 글쓰기 연습(`writing_practice.html`), 숫자 색칠(`color_by_number.html`).

### 4단계 폴백 전략과 서킷 브레이커

AI 서비스의 안정성을 보장하기 위해 **4단계 폴백 아키텍처**를 적용한다.

- **Tier 1**: Haiku 4.5 + 구조화 출력 → 동화 맞춤 콘텐츠 생성
- **Tier 2**: Haiku 4.5 + 단순화된 프롬프트 → 기본 콘텐츠 생성 (Tier 1 실패 시)
- **Tier 3**: 사전 제작 기본 템플릿 → 활동 유형 + 연령으로 선택 (API 실패 시)
- **Tier 4**: 정적 비상 PDF → 범용 "오늘의 활동" 시트 (모든 것 실패 시)

Claude API 호출에는 **pybreaker** 라이브러리로 서킷 브레이커를 적용한다. 연속 5회 실패 시 회로를 열고, 60초 후 반열림(half-open) 상태에서 테스트 요청을 보낸다. **일시적 오류만 카운트**(429, 500, 502, 503, 타임아웃)하고, 400(잘못된 요청)이나 401(인증)은 제외한다. 성공한 AI 응답을 Redis에 캐싱하여 회로가 열린 동안 스테일 캐시를 서빙하는 전략도 유효하다.

## 오픈소스 리소스 스택: 폰트, 일러스트, 생성 알고리즘

### 한글 교육용 폰트 3종 추천

| 폰트명 | 제작자 | 라이선스 | 용도 |
|--------|-------|---------|------|
| **학교안심 둥근미소** | KERIS(한국교육학술정보원) | SIL OFL | ★ 제목·활동 지시문 (28~36pt) — 둥글고 친근한 교육 전용 폰트 |
| **윤초록우산 어린이체** | 초록우산 어린이재단 × 윤디자인 | 무료 상업 사용 | ★ 어린이 활동지 본문 (18~24pt) — 어린이 전용 설계 |
| **Noto Sans KR** | Google × Adobe | SIL OFL | 본문·레이블 텍스트 (18~20pt) — 6가지 웨이트, 뛰어난 가독성 |

보조 폰트로 나눔손글씨(네이버, SIL OFL)를 동화 텍스트에, 강원교육 모두체(강원도교육청, 무료)를 부제목에 활용할 수 있다. 폰트 탐색은 **눈누(noonnu.cc)**에서 라이선스별 필터링이 가능하다.

### 알고리즘 활동 생성 라이브러리

미로 생성은 **labyrinth**(Python, DFS/Kruskal/Prim/Wilson, GUI 시각화, 이미지 출력)가 가장 성숙하다. 단어 찾기(Word Search)는 **word-search-generator**(PyPI v5.0, 난이도 조절, 방향 제어, 마스크 모양)가 적합하며 한글 격자 적응이 필요하다. 십자말 풀이는 **genxword**(다국어 지원, PDF/SVG 출력)가 한글에 가장 적합하다. 점잇기는 전용 라이브러리가 없으나 **svgpathtools**로 SVG 경로를 따라 등간격 점을 배치하는 방식으로 구현한다. 다른 그림 찾기 역시 전용 도구가 없으나 svgwrite/lxml로 장면을 복제한 뒤 N개 요소를 프로그래밍 방식으로 변경하는 접근이 가능하다.

### 인쇄용 활동지 디자인 가이드라인

만3~5세 활동지는 **A4(210×297mm), 여백 15~20mm, 인쇄 해상도 300DPI**가 기본이다. 폰트 크기는 본문 최소 **18~24pt**, 제목 28~36pt. 산세리프 폰트를 사용하고, 줄간격은 폰트 크기의 120~150%. 활동 요소의 선 두께는 **최소 2~3pt**, 미로 통로는 충분히 넓게. 색상은 따뜻하고 밝은 채도 높은 색을 사용하되, 흑백 인쇄에서도 활동이 작동하도록 설계한다. 텍스트 대비비 최소 4.5:1.

## 결론: 구현 로드맵과 핵심 설계 원칙

이 리서치의 결론은 세 가지 핵심 원칙으로 수렴한다. **첫째, 템플릿이 왕이다.** AI는 텍스트 콘텐츠만 생성하고, 레이아웃·시각·인쇄 품질은 사전 검증된 HTML/CSS/SVG 템플릿이 전적으로 담당해야 한다. 이것이 안정성과 비용 효율성을 동시에 달성하는 유일한 경로다. **둘째, 누리과정 태깅이 시장 진입 무기다.** 모든 활동지에 5개 영역·내용범주 태그를 자동 부여하면, 교사의 보육일지 작성과 어린이집 평가제 대응을 직접 지원하는 실질적 가치를 제공하게 된다. 경쟁 플랫폼(키드키즈, 꼬망세)이 수동 태깅하는 것을 AI가 자동화하는 차별점이다. **셋째, 독후활동지는 이미 검증된 카테고리다.** 찬진교육 유아스토리가 독후활동지를 별도 주요 카테고리로 운영할 만큼, 동화 기반 활동지는 한국 유아교육 현장에서 확립된 수요가 있다.

구현 우선순위는 다음과 같다. **Phase 1 (MVP, 2~3주)**: 색칠하기·선긋기(만3세) + 미로·따라쓰기(만4세) + 순서 맞추기·글쓰기(만5세) 총 6종 템플릿. Haiku 4.5 파이프라인 구축. WeasyPrint + 학교안심 둥근미소 폰트. **Phase 2 (4~6주)**: 전체 10종 활동 유형 커버. 알고리즘 미로/점잇기 생성기 통합. 프롬프트 캐싱 + Redis 응답 캐싱. 서킷 브레이커 + 4단계 폴백. **Phase 3 (7~10주)**: 혼합반 3단계 자동 분류. QA 에이전트 루프백. 배치 API 사전 생성. SVG 클립아트 라이브러리 200종 큐레이션 완료. 활동지당 예상 비용 $0.003~0.007, 월 10,000매 기준 $21~42.
